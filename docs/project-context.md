Many students experience delays and confusion when looking for official university forms on the school website because documents are scattered, difficult to locate, or not grouped clearly by office and purpose. This case study proposes a Student Document Repository System that centralizes frequently used student-related forms such as subject withdrawal, INC completion, and course shifting requests in one searchable web platform. The system is intended for students as the primary users, with support for administrators who manage form records. Using a MongoDB NoSQL backend and a web-based interface, the application provides CRUD operations for form management, keyword-based search, dashboard visualization of stored data, and a JSON backup download feature for data safety and portability. By organizing form access into a single digital process, the system improves efficiency, accessibility, and user experience for students handling academic transactions.
